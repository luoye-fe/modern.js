#Testing

This chapter describes the test-related configuration

TIP

You need to enable the unit testing feature with pnpm run newfirst.

#jest

  • Type: Object | Function
  • Default value: {}

The configuration corresponding to Jest, when of type Object, can be configured with all the underlying configurations supported by Jest .

modern.config.ts

import { defineConfig } from '@modern-js/module-tools';
export default defineConfig({
testing: {
jest: {
testTimeout: 10000,
},
},
});

When the value is of type Function, the default configuration is passed as the first parameter and a new Jest configuration object needs to be returned.

modern.config.ts

import { defineConfig } from '@modern-js/module-tools';
export default defineConfig({
testing: {
jest: options => {
return {
... . options,
testTimeout: 10000
}
}
}
});

#transformer

  • type: 'babel-jest' | 'ts-jest'
  • Default value: 'babel-jest'

Configure the compilation tool for the source code when executing tests: babel-jest or ts-jest. The default is babel-jest.

INFO

babel-jest can also compile TS files without type errors, so use ts-jestif you need to check the TS type when running tests.
